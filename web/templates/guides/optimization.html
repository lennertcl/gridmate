Optimization Guide

How to set up EMHASS energy optimization — Go to configuration

Field Explanations

Prerequisites

Before configuring optimization, ensure the following are set up:

  • EMHASS add-on: Install and start EMHASS from the HA Add-on Store
  • Energy contract: Configure your energy contract
  • Solar panels: If applicable, configure in solar panel settings
  • Home battery: If applicable, add a device with the home_battery type in device settings and enable opt_enabled to include the battery in optimization.
  • Deferrable devices: Add the Deferrable Load type to devices in device settings

EMHASS Connection

EMHASS URL
REST API URL of the EMHASS add-on. Can be left blank for autofill.
Actuation Mode
Manual: show schedule only.
Automatic: control devices directly. (Not working yet, to be added later)
Day-Ahead Run Time
Time of day to run the day-ahead optimization (e.g., 05:30 before morning consumption starts).
Not working yet, to be added later. You can use a HA automation to POST to the /api/optimization/run endpoint for now
Enable Optimization
Master switch for automated runs. Manual optimization is always available from the dashboard.

Grid & Load

Max Grid Import (W)
Maximum power to pull from the grid. The optimizer will respect this limit.
Max Grid Export (W)
Maximum power to feed to the grid. Set to your inverter's max output if no export limit applies.
Load Power Source
The base load power expected from devices not included in the optimization. E.g. you may expect a higher base load between 18:00-20:00 if that's when you cook dinner.
Choose how non-deferrable household load is provided to the optimizer:
  • Home Assistant Sensor: A sensor entity reporting total household power in Watts, excluding deferrable loads. EMHASS uses this as sensor_power_load_no_var_loads.
  • Fixed Schedule: Define time blocks with a constant power value each. The schedule repeats daily and is sent to EMHASS as load_power_forecast at runtime.

Battery

Battery optimization is controlled per-device. In device settings, enable the opt_enabled parameter on a device with the home_battery type. Battery SOC limits (min, max, target) and efficiency settings are configured as parameters on the device itself. Note that EMHASS currently only supports a singly battery in optimization.

EMHASS Configuration

GridMate pushes a complete configuration to EMHASS via the set-config API whenever you save settings. Use the Fetch Config button to view the config currently active in EMHASS. Runtime parameters (energy cost forecasts, load power forecasts, current battery SOC) are passed automatically by GridMate during each optimization call.
