Optimization Guide

How to set up EMHASS energy optimization — Go to configuration

Field Explanations

Prerequisites

Before configuring optimization, ensure the following are set up:

  • EMHASS add-on: Install and start EMHASS from the HA Add-on Store
  • Energy contract: Configure your energy contract
  • Solar panels: If applicable, configure in solar panel settings
  • Home battery: If applicable, add a device with the home_battery type in device settings and enable opt_enabled to include the battery in optimization.
  • Deferrable devices: Add the Deferrable Load type to devices in device settings
  • REST command (for automatic mode): Add the following to your Home Assistant configuration.yaml and restart HA: 
    rest_command:
  gridmate_run_optimization:
    url: "{{ addon_url }}/api/optimization/run"
    method: POST
    content_type: "application/json"
    payload: '{}'
    This allows GridMate to create a HA automation that triggers daily optimization at your configured time.
    The URL above is auto-detected from your GridMate installation. It should match the hostname visible in your browser when you open GridMate (e.g. http://homeassistant.local:8123/5b918bf2_gridmate means the addon hostname is 5b918bf2-gridmate, underscore replaced by dash).

EMHASS Connection

EMHASS URL
REST API URL of the EMHASS add-on. Can be left blank for autofill.
Actuation Mode
Manual: Show the optimized schedule on the dashboard only. No devices are controlled.
Automatic: GridMate creates Home Assistant automations that turn devices on/off at the scheduled times. After each optimization run, one automation is created per device with time-based triggers matching the schedule. These automations are visible in HA under Settings → Automations with a [gridmate] prefix. They are automatically cleaned up and recreated on every optimization run.
For devices with variable power (Constant Power disabled), the automation also sets the target power via the Power Control Entity.
Day-Ahead Run Time
Time of day to run the day-ahead optimization (e.g., 05:30 before morning consumption starts). When optimization is enabled, GridMate creates a [gridmate] Daily Optimization automation in HA that triggers at this time. This requires the rest_command prerequisite above.
Enable Optimization
Master switch for automated runs. Manual optimization is always available from the dashboard. Disabling this will remove all [gridmate] automations from Home Assistant.

Grid & Load

Max Grid Import (W)
Maximum power to pull from the grid. The optimizer will respect this limit.
Max Grid Export (W)
Maximum power to feed to the grid. Set to your inverter's max output if no export limit applies.
Load Power Source
The base load power expected from devices not included in the optimization. E.g. you may expect a higher base load between 18:00-20:00 if that's when you cook dinner.
Choose how non-deferrable household load is provided to the optimizer:
  • Home Assistant Sensor: A sensor entity reporting total household power in Watts, excluding deferrable loads. EMHASS uses this as sensor_power_load_no_var_loads.
  • Fixed Schedule: Define time blocks with a constant power value each. The schedule repeats daily and is sent to EMHASS as load_power_forecast at runtime.

Battery

Battery optimization is controlled per-device. In device settings, enable the opt_enabled parameter on a device with the home_battery type. Battery SOC limits (min, max, target) and efficiency settings are configured as parameters on the device itself. Note that EMHASS currently only supports a singly battery in optimization.

Automatic Device Control

When actuation mode is set to Automatic, GridMate manages Home Assistant automations to control your devices according to the optimized schedule:

How it works
After each optimization run, GridMate creates one HA automation per managed device. Each automation uses time-based triggers to turn the device on and off at the scheduled times. These automations appear in HA under Settings → Automations with a [gridmate] prefix in their name.
Control Entity
Every device that should be controlled automatically needs a Control Entity configured (via the Automatable Device type). This is the HA entity (e.g. switch.boiler) that will be turned on/off.
Power Control Entity
For devices with Constant Power disabled (variable power optimization) or home batteries, set the Power Control Entity to a number.* HA entity. The automation will call number.set_value to set the target power level for each schedule block. This is useful for EV chargers, batteries, or other devices where the power level can be adjusted.
Cleanup
All [gridmate] device automations are deleted and recreated fresh on every optimization run, ensuring the schedule is always up to date. Disabling optimization removes all [gridmate] automations. Switching to manual mode removes device automations but keeps the daily trigger.

Weekly Device Schedule

The weekly schedule lets you define per-day optimization settings for each deferrable load device. Configure exactly which days each device should run, how many cycles per day, and optionally override the time window.

Schedule Table
The table shows devices as rows and days of the week (Mon–Sun) as columns. Each row includes an opt_enabled toggle (the master switch for including the device in optimization). Click any cell to edit the schedule for that device on that day. Each cell shows a badge with the number of cycles: green when the device is active, grey when disabled.
Cycles
How many times the device should be powered on that day (0–5). Setting cycles to 0 disables the device for that day. When set to more than 1, the device’s time window is split into non-overlapping sub-windows so EMHASS schedules each cycle separately.
Hours Between Runs
Minimum gap (in hours) between consecutive cycles. Only relevant when cycles is greater than 1. If the time window is too short to fit all cycles with the requested gap, GridMate falls back to a single cycle.
Earliest Start / Latest End
Optionally override the device’s default time window for a specific day. The fields are pre-filled with the device defaults. Leave unchanged to use the device’s configured values.
Apply Button
Click Apply in the cell editor to confirm your changes. Changes are not saved until you click Apply.
Apply to All Days
Use the button to copy Monday’s configuration to all other days of the week for that device.
No Schedule Configured
If no weekly schedule is set, every enabled device defaults to one cycle per day using its full time window. This preserves backward compatibility.

Dashboard Overrides

The optimization dashboard shows today’s schedule and lets you override the next optimization without changing the weekly schedule permanently.

Today’s Schedule
Each managed device card shows the number of cycles and gap configured for today (from the weekly schedule). If no schedule is set, it defaults to one cycle.
Override Next Optimization
Click Override next optimization on a device card to temporarily change its settings for the next optimization run only. You can change the number of cycles (set to 0 to disable), adjust the gap between runs, and override the earliest start and latest end times. The same editor fields are used as on the settings page. The override is shown as a yellow banner on the device card.
Clearing Overrides
Click the × button on the override banner to remove it. Overrides are also cleared automatically after the next optimization run completes.

Optimization Dashboard Chart

The Energy Plan chart on the optimization dashboard combines both power and price context for the selected run.

Power plan
Base load, managed devices, battery charging/discharging, and solar production show how EMHASS plans energy flows across the horizon.
Price lines
The chart also overlays the import and export prices that were actually sent to EMHASS for that optimization run. This makes it easier to understand why loads were shifted to certain periods.

EMHASS Configuration

GridMate pushes a complete configuration to EMHASS via the set-config API whenever you save settings. Use the Fetch Config button to view the config currently active in EMHASS. Runtime parameters (energy cost forecasts, load power forecasts, current battery SOC) are passed automatically by GridMate during each optimization call.
